Live2D模型加密保护软件 - Demo阶段功能需求文档


项目背景与目标


本项目旨在开发一套 Live2D模型加密保护系统，以防止Live2D模型资源在使用过程中被非法复制和盗用，从而保护我方模型资产的知识产权。项目将分为两个阶段推进：


验证性Demo开发阶段（当前阶段）：构建一个具备核心功能的Demo，验证加密模型在VTube Studio（以下简称VTS）中可被正常加载；


正式系统开发阶段：基于Demo成果，开发完整的加密管理系统。




说明：




VTS 是一款可通过Steam免费下载的Live2D模型加载软件；


Live2D模型 是指由Cubism Editor制作的可动插画资源，广泛应用于虚拟直播、游戏等场景；


VTS本身并不具备加密或版权保护机制，模型资源在使用过程中易被明码复制，本项目的目标是通过加密机制，防止非法使用和传播，而非破解或修改VTS本体。


目前市面上已有部分具备类似功能的加密工具和插件，但本项目将独立开发属于我方的解决方案。相关竞品软件将作为功能参考（具体名称和使用细节可私下沟通）。本项目为个人主导项目，将采用分阶段迭代方式进行，目前处于第一阶段的Demo开发。


Demo开发目标


构建一个具备核心功能的加密解密演示程序，确保通过该程序启动后，VTS能够读取已加密的Live2D模型，且整个使用过程中，模型文件未以明文形式暴露。


该程序将在后续正式开发中作为关键功能组件使用，因此需预留开发接口，并配套简要的接口说明文档。


核心功能需求（初步设想）


以下为预计实现的功能模块，具体实现方案可与开发者在实际开发过程中进一步讨论确认。


1. 模型加密模块


实现基本的模型文件加密与解密功能；


加密文件应确保无法被VTS或其他软件直接识别或读取；


加密文件在通过本程序加载后，VTS能够正常读取模型内容；


提供基础的密钥生成与管理机制。


2. 虚拟文件系统集成


使用 Dokan 库，实现基于用户态的虚拟文件系统；


可挂载为虚拟磁盘（如 X: 盘）；


实现模型文件的透明映射、读取请求拦截与转发；


保证除VTS外，其他软件无法访问明文模型内容。


3. 进程识别机制


实现对目标进程（VTS）的进程ID获取与验证；


支持仅允许特定进程（VTS）访问虚拟磁盘内容；


支持“两阶段启动流程”：


a) 启动程序，获取VTS进程信息；


b) 重启VTS，通过挂载后路径加载模型资源。


4. 文件映射机制


加密模型文件在虚拟磁盘中应可映射为正常模型路径；


支持路径重定向、符号链接等机制；


实现一定程度的路径伪装，增强安全性。


Demo完成标准


成功实现Live2D模型加密，并通过程序加载后可被VTS正常识别；


加密后的模型文件无法被直接加载或拷贝使用；


实现基础的进程验证机制，确保仅VTS能访问虚拟磁盘资源；


程序具备基本加解密接口与使用说明，便于后续扩展与集成；


整个流程中，模型资源未在磁盘以明文形式存储或暴露。

熟悉基本加密算法实现、密钥管理机制；


熟练掌握 进程间通信、进程识别与管理；


精通 C++ / C# 开发语言；


有系统级钩子（Hook）开发经验。

熟悉VTS与Live2D模型文件结构；


具备安全编程和反调试意识。
